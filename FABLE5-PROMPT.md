# Fable 5 実装プロンプト — スマホ・ポッドキャスト収録アプリ（Anchor復刻・MVP）

このファイルをFable 5に渡して実装させる。以下をそのまま指示として使ってよい。

---

## 目的

Spotifyが終了させた旧アプリ「Anchor」の**収録・編集・書き出し**部分を、スマホのブラウザだけで動く超シンプルなウェブアプリとして復刻する。個人利用。

## 形式・技術方針（厳守）

- **単一のウェブアプリ**。iPhone / Android の Safari・Chrome で URL を開くだけで動く。
- **インストール・サーバー・審査・ログイン、すべて不要**。
- 収録・編集・書き出しは**すべてブラウザ内（クライアント側）で完結**。ネット接続は収録中不要。
- **Xcode / ネイティブ / App Store は使わない。**
- 使う技術：`MediaRecorder`（マイク収録）、`Web Audio API`（BGMミックス・書き出し）、素の HTML/CSS/JS。フレームワークは軽量なら可、無しでも可。
- できれば**ファイル最小構成**（1〜数ファイル）。ローカルで開いても動くこと。
- スマホ縦画面前提。指で押しやすい大きめUI。

## 作る機能（この4つだけ）

### 1. スマホ収録
- 大きな録音ボタン。押すと録音開始、もう一度で停止。
- **一時停止・再開**できる。
- 録音した音声は「**チャンク**（かたまり）」として1本ずつリストに積まれる。
  - 例：3分喋る＝チャンク1。もう一度録る＝チャンク2。
- 各チャンクは録音時間を表示。再生ボタンで試聴できる。
- 収録中は「録音中」が明確にわかる表示（赤丸・経過時間など）。

### 2. チャンク並べ替え
- チャンクのリストを**ドラッグで並べ替え**できる。
- 各チャンクを**削除**できる。
- （任意・できれば）チャンク先頭/末尾の簡単なトリミング。難しければ無しでよい。

### 3. BGM挿入
- **BGMもチャンクとしてリストに挿入**する（Anchor方式）。
- BGMの入手方法は2つ用意：
  - (a) **ユーザーが手持ちの音声ファイルをアップロード**して挿入（最優先。これだけでも可）。
  - (b) （任意）ロイヤリティフリーのサンプルBGMを数個内蔵。
- BGMチャンクも並べ替え・削除の対象。
- （任意・できれば）BGMを「トークの背後に重ねる（ダッキング）」モード。難しければ**まずは直列つなぎ（BGMも1チャンクとして順番に流す）だけでよい**。

### 4. 書き出し
- 全チャンクを**順番どおり1本の音声ファイルに結合**して書き出す。
- 形式：**まず確実に鳴るものを優先**（WAV でも可。可能なら MP3 / m4a）。
- 書き出したファイルを**スマホにダウンロード保存**（`<a download>` 等）。ユーザーはそれを Spotify for Creators に自分で手アップロードする。

## 作らない機能（明確に除外）

- RSS配信・自動アップロード・Spotify API連携（← 将来のBプラン。今回なし）
- 収益化・投げ銭・広告・サブスク
- 分析・統計
- リモート共同収録・リスナー音声投稿
- Music + Talk（Spotify楽曲の埋め込み）
- ログイン・アカウント・クラウド保存

## 既知の制約（許容する。回避策を作り込まない）

- **iOS Safari はバックグラウンド収録が止まる** → 「収録中は画面を消さないで」の注意書きを1行出すだけでよい。
- 初回にマイク許可ポップアップが出る → 正常。
- 精密な波形編集・音質補正は**対象外**。

## UIの想定（目安。実装者裁量で改善可）

1. 上部：アプリ名 + 「収録中は画面を消さないでください」の注意。
2. 中央：大きな録音ボタン（●）＋経過時間。一時停止ボタン。
3. その下：チャンクのリスト（並べ替え・再生・削除）。各チャンクにトーク/BGMのアイコン区別。
4. 「BGMを追加」ボタン（ファイル選択）。
5. 最下部：「書き出す」ボタン。押すと結合してダウンロード。

## 完成の判定（これができたら完成）

1. スマホのブラウザでURLを開ける。
2. マイクで複数チャンク録音できる。
3. チャンクを並べ替え・削除できる。
4. 手持ちBGMファイルを挿入できる。
5. 全部を1本に結合して書き出し、スマホに保存できる。
6. 保存した音声が普通に再生できる。

---

## 実装の進め方（Fable 5への指示）

- まず**動く最小版**（収録→リスト→結合書き出し）を1つ作り、実機のスマホブラウザで鳴らして確認してから、BGM・並べ替えを足す。
- 各ステップで実際にスマホで音が出るかを最優先で確認する。
- 迷ったら「シンプル・確実に鳴る」を選ぶ。凝った処理より完走を優先。
